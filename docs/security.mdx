## Key handling

- Mnemonic payloads are stored encrypted at rest
- Decryption occurs only at runtime for signing flows
- Seed phrases and private keys must never be logged or committed

## Policy controls

Caishen supports controls including:

- per-tx / per-hour / per-day spending limits
- recipient allow/block lists
- chain allow/block lists
- active time windows
- global pause/resume

## Logging

Activity logs are append-only JSON lines under `~/.caishen/activity.log` for auditable traces.

## Operational recommendations

- Use dedicated runtime users for production
- Keep strict filesystem permissions for runtime config
- Require human confirmation for all value-moving actions
- Prefer quote/preflight before execute

CLI enforcement examples:

```bash
caishen transfer --token USDT --chain ethereum --to 0xRecipientAddress --amount 1 --dry-run
caishen transfer --token USDT --chain ethereum --to 0xRecipientAddress --amount 1 --confirm "CONFIRM TRANSFER 1 USDT ON ethereum TO 0xrecipientaddress SIG <FROM_PREFLIGHT>"

caishen protocol --chain ethereum --type swap --label velora --method quote --dry-run
caishen protocol --chain ethereum --type swap --label velora --method quote --confirm "CONFIRM PROTOCOL swap.velora.quote ON ethereum SIG <FROM_PREFLIGHT>"
```

For detail, see [SECURITY.md](../SECURITY.md) and [POLICY.md](../POLICY.md).
